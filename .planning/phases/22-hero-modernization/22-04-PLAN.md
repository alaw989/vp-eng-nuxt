<task type="auto">
  <name>Task 1: Implement content entrance animations with Vue transitions</name>
  <files>components/HeroStatic.vue</files>
  <action>
Add smooth entrance animations to hero content using Vue's Transition component:

1. Wrap content in Transition component:
   ```vue
   <Transition
     name="hero-content"
     mode="out-in"
     :css="!prefersReducedMotion"
     appear
   >
     <div
       key="hero-content"
       class="container text-center text-white"
     >
       <h1 class="hero-content-item">{{ headline }}</h1>
       <p class="hero-content-item">{{ subheadline }}</p>
       <NuxtLink v-if="showCta" class="hero-content-item">{{ ctaText }}</NuxtLink>
     </div>
   </Transition>
   ```

2. Add CSS transitions:
   ```css
   /* Fade-in and slide-up animation */
   .hero-content-enter-active {
     transition: all 0.6s cubic-bezier(0.16, 1, 0.3, 1);
   }

   .hero-content-leave-active {
     transition: all 0.3s ease-in;
   }

   .hero-content-enter-from {
     opacity: 0;
     transform: translateY(40px);
   }

   .hero-content-leave-to {
     opacity: 0;
     transform: translateY(-20px);
   }

   /* Staggered timing for child elements */
   .hero-content-enter-active .hero-content-item:nth-child(1) {
     transition-delay: 0ms;
   }

   .hero-content-enter-active .hero-content-item:nth-child(2) {
     transition-delay: 100ms;
   }

   .hero-content-enter-active .hero-content-item:nth-child(3) {
     transition-delay: 200ms;
   }

   /* Respect prefers-reduced-motion */
   @media (prefers-reduced-motion: reduce) {
     .hero-content-enter-active,
     .hero-content-leave-active {
       transition: opacity 0.3s ease;
     }

     .hero-content-enter-from {
       opacity: 0;
       transform: none;
     }

     .hero-content-leave-to {
       opacity: 0;
       transform: none;
     }
   }
   ```

3. Animation guidelines (from research):
   - Duration: 300-600ms (hero entrance can be longer than micro-interactions)
   - Easing: cubic-bezier(0.16, 1, 0.3, 1) for natural feel
   - Stagger: 100ms between elements for cascading effect
   - Direction: Fade-in + slide-up (40px) for content entrance

4. Reduced motion support:
   - Check prefersReducedMotion computed ref (from parallax task)
   - When reduced motion: only fade opacity, no transform
   - Shorter duration (300ms) for reduced motion
   - Preserve staggered timing even with reduced motion

5. Performance optimization:
   - Use opacity and transform only (GPU accelerated)
   - Avoid animating layout properties (width, height, margin)
   - Use will-change sparingly (not needed for one-time entrance)
   - CSS-only approach (no JavaScript animation libraries)

6. Cross-fade for content changes (if variant changes):
   - Use mode="out-in" for smooth transitions between variants
   - 600ms exit, 300ms entrance (faster exit feels snappier)
  </action>

1. Content fades in smoothly on page load
2. Staggered timing visible (headline, then subheadline, then CTA)
3. Animations disabled when prefers-reduced-motion enabled
4. No jarring motion or flashing
  </verify>
  <done>Hero content has smooth entrance animations with staggered timing</done>

2. Hover effect guidelines:
   - Duration: 300ms (matches micro-interaction best practices)
   - Transform: -translate-y-1 (subtle lift, not jarring)
   - Shadow: lg to xl (depth increase on hover)
   - Icon: Arrow moves slightly (suggests forward motion)
   - Background: Gradient shift (visual feedback)

3. Focus visible state:
   - High contrast outline (2px white)
   - Outline offset for spacing
   - Ring for additional visibility

4. Reduced motion support:
   ```css
   @media (prefers-reduced-motion: reduce) {
     .cta-button {
       transition: background-color 300ms ease, box-shadow 300ms ease;
       transform: none !important;
     }

     .cta-button:hover {
       transform: none;
     }
   }
   ```

5. Touch device consideration:
   - Hover effects work on tap (mobile)
   - No hover effects on devices that don't support hover
   - Use @hover pseudo-class for hover-only devices
